# coldstart-mcp

Lightweight navigation layer for AI agents. Answers one question: which files are relevant to this task? Fast static index over file paths, symbol names, and exports â€” queried via two operations, `find` (ranked file paths matched against declared names: filenames, path segments, exported symbols, plus a repo-wide grep-recall pass) and `gs` (for a single file: top-level symbols with per-symbol cross-file callers, 1-hop internal imports, and reverse importers; `view: "full"` is the default and returns all of it in one call). Both are exposed two ways with identical output: as **CLI commands** (`coldstart find` / `coldstart gs`) for shell-capable agents â€” the primary path â€” and as **MCP tools** (`find` / `gs`) for no-shell clients like Claude Desktop.

**Architecture â€” one keeper, thin readers.** The query surfaces are stateless readers over an on-disk cache; a single background **keeper** process keeps that cache fresh. It does NOT serve queries.
- **Keeper** (`coldstart --daemon`, `src/index.ts` runKeeper + `src/index-manager.ts`): watches the repo, patches/rebuilds the index, debounce-saves it to the disk cache. Lazy-spawned by the first reader (`src/keeper.ts` ensureKeeper); logs to `~/.coldstart/daemon/<root>.log`; exits when its lockfile is removed. The lock carries pid/rootDir/version (no port â€” it serves nothing).
- **CLI readers** (`coldstart find`/`gs`, `src/cli.ts`): load the cache under a load profile, run the same engine, print, exit. They `ensureKeeper` so uncommitted edits stay live. **Readers NEVER build**: cache miss â†’ wait for the keeper's first save (180 s); git-HEAD drift â†’ wait for its reconcile re-save (12 s); in-process build only on the no-keeper fallback.
- **MCP reader** (default invocation, `src/server/mcp.ts` + `src/index.ts` makeCacheReader): a long-lived stdio server that reads the keeper's cache and mtime-reloads when it changes. The keeper is the single freshness authority; the server has no watcher of its own.

**Freshness (no TTL â€” deleted in v18).** Validity = CACHE_VERSION + git HEAD + reconcile + live watcher. At keeper start, `src/indexer/reconcile.ts` stat-walks every indexed file against its `[mtimeMs,size]` fingerprint (stamped in baseIndexedFile) + git-diffs against the indexed HEAD, then patches exactly the changed set (branch switch = ~3 s patch, was a full rebuild). After every patch, `src/indexer/invariants.ts` lints the index (violation â†’ auto-rebuild); after every save, a rotating 50-file fingerprint audit re-patches watcher-missed drift. `keeper-state.json` (last reconcile/patch/rebuild/save) + `repair.jsonl` (failures) sit beside the cache; `status` renders both. The keeper watches its own lockfile with pid-ownership + a 30 s poll backstop (foreign pid â†’ exit).

**`find` speed:** the repo-wide reference scan resolves a real ripgrep (`src/server/searcher.ts`: COLDSTART_RG â†’ PATH â†’ bundled @vscode/ripgrep â†’ editor-app copies; winner persisted in `~/.coldstart/searcher.json`), falling back to git grep â†’ grep â†’ Node scan.

For Rails repos: the Ruby parser emits synthetic edges for `has_many`/`belongs_to`/`has_one`/`has_and_belongs_to_many` (gated to `app/models/`), parses `config/routes.rb` resources, and adds bidirectional controllerâ†”views edges. Polymorphic associations and gem-backed models stay unresolved (runtime DSL).

**Index pipeline:** walk â†’ parse (Tree-sitter via **web-tree-sitter (WASM)** â€” the native `node-tree-sitter` engine was dropped in #68; all 15 grammars are inert `.wasm` vendored in `vendor/wasm/`, loaded once by `ensureParsersReady()` in `src/indexer/parser.ts`, no per-grammar peer-dep / node-gyp / install scripts â†’ plain `npm i` for TS/JS/JSX/TSX/Java/Ruby/Python/Go/Rust/C#/PHP/Kotlin/C++/YAML/TOML/XML/Groovy; SFC script blocks extracted from Vue/Svelte/Astro before TS parsing; GraphQL/.env/AngularJS 1.x via regex extractors; Swift/Dart not indexed) â†’ resolve imports â†’ build graph (including cross-file call edge resolution) â†’ save cache. Behaviour is byte-identical to the old native engine (proven with natives pruned) and 1.3â€“3.6Ă— faster through the walk-heavy extract phase.

**Live updates:** the keeper's `fs.watch` listener keeps the cache current. Changes are debounced (400 ms), then either patched incrementally (â‰¤ max(30, 20% of repo) files, ~2â€“5 ms/file) or trigger a full background rebuild, and the cache is re-saved ~5 s after edits settle â€” in atomic generations (`g<N>-` segments; `meta.json` names the gen and commits last; mixed-generation reads impossible). Patch mirrors the walker's dir rules (hidden/excluded dir segments rejected â€” the watcher/porcelain can feed paths the walker never would). No restarts required.

**Notebook (`src/kb/`):** `coldstart kb search|lookup|write|commit|status|lint|render|view|init|migrate` (`view` = single-file self-contained HTML browser of the notebook, generate-and-open, freshness stamped at generation time; `src/kb/view.ts` + `view-template.js`, human-only â€” never an MCP tool). The read/write surface is ALSO exposed as MCP tools with identical output â€” `kb_search`/`kb_lookup`/`kb_write`/`kb_status` in `src/server/mcp.ts` (the MCP server is a *writer* for `kb_write`) â€” for no-shell clients; `kb commit` is CLI/human-only, never an MCP tool. `.coldstart/notebook/` holds per-note append-only `.raw/<id>.jsonl` logs (source of truth, committed; a note = pure fold of its log) + derived md notes. kb readers NEVER load the code index â€” the keeper derives a `kb-notes.json` sidecar (anchor freshness stamps) single-flight at watch start / `.raw` batches / post-save. Two-phase `kb write` (exit 3 = candidates â†’ `--into <id>` or `--new`). Concurrency-safe multi-session writes: freshly-coined ids created O_EXCL (coin race â†’ EEXIST â†’ re-coin; never a silent merge), the fold UNIONS anchor symbols, derived md is temp+rename atomic, `kb write -` reads stdin as a stream (regressions: `tests/kb-store-atomic.test.ts`, `tests/kb-concurrent-create.test.ts`). `kb commit` is the one sanctioned git path (pathspec-commits only `.raw/` + skeleton â€” notes never ride in feature commits). Hooks: `hooks/kb-elicit.mjs` (Stop/SubagentStop; v5 TRIGGER-TIMED capture â€” per-file evidence records `hooks/evidence.mjs` (edit/read/gs tiers, mentions never count, `.coldstartignore` filtered at the root via `hooks/ignore.mjs`) feed `hooks/trigger.mjs` (score=reads+2Ă—settled-edits+active-stops, fresh-noted=0; arm@10, fire on descent (1 quiet stop) or capâ†’non-blocking pending file, .git-HEAD driftâ†’blocking; surge removed in #85; NO first-stop/time/classification); payload = `hooks/capture-payload.mjs` v5 checklist (worklist-is-scope, product-level flow gate, graph-blind-spot usage rule via `coldstart consumers`); SubagentStop stays one-shot blocking, derives `<transcript-stem>/subagents/agent-<aid>.jsonl`) + `hooks/kb-recall.mjs` (UserPromptSubmit; delivers any pending capture payload FIRST, then strong title/alias/anchor match â†’ compact title+gist block, else silence) â€” wired by `coldstart init` for Claude Code, Codex, and Cursor (the `kb init` alias re-wires just the notebook). Contract: `docs/notebook-kb-design.md`.

**Setup:** `coldstart init` asks two things â€” **experience** (`cli`/`mcp`, also via `--experience`) and **client** (`claude`/`cursor`/`codex`/`other`, also via `--client`; never auto-detected) â€” then writes the agent guidance (CLI- or MCP-flavored) â€” as `coldstart.md` for Claude Code and Other, inlined into the client's own rules file for Cursor and Codex (neither resolves `@file` refs) â€” and wires the chosen client (`src/init.ts`). **Claude Code:** `coldstart.md` + `@coldstart.md` into CLAUDE.md + Claude-specific navigation and notebook hooks in `.claude/settings.json` (mcp experience also writes `.mcp.json`). **Codex:** the full guidance inlined in a marked block in AGENTS.md (no separate coldstart.md) + separate Codex navigation and notebook hooks in `.codex/hooks.json`; its capture hook parses Codex rollout/subagent transcripts (mcp experience also writes `[mcp_servers.coldstart]` into `.codex/config.toml`). **Cursor:** `.cursor/rules/coldstart.mdc` carrying the full guidance inline (Cursor doesn't resolve `@file` refs in rules) + Cursor navigation and notebook hooks in `.cursor/hooks.json` (`preToolUse` dedup-deny, `postToolUse` nudge, `beforeSubmitPrompt` recall, `stop`/`subagentStop` capture; the capture hook parses Cursor's own conversation JSONL scoped to the current turn via `turn_ended`; recall rides `beforeSubmitPrompt.additional_context`, which Cursor honors despite its docs; capture is loop-guarded on `loop_count===0` since `followup_message` re-fires `stop`) + (mcp) `.cursor/mcp.json`. **Other:** coldstart.md + printed wiring directions. All writers merge idempotently and point hooks/MCP at the running install (`installRoot()` in `src/init.ts`, derived from the module path â€” no `~/.coldstart/versions/` copy), so `npm update` is picked up automatically (`npm uninstall` makes the wired hooks no-op but leaves the entries/files behind). Platform hook entrypoints stay separate; Cursor/Codex/Claude share the protocol-neutral detector core â€” only input adaptation, the transcript walk, and the output envelope differ per host.

**Unwire (`coldstart unwire`, `src/unwire.ts`, #69):** the explicit reverse of `init` (npm fires no reliable pre/postuninstall, and a global uninstall has no registry of inited repos). Reuses `init`'s own idempotency detectors (`isColdstartHookEntry`/`isKbHookEntry`/`isCodexHookEntry`/`isCursorHookEntry`/`stripCodexColdstartTable`, exported from `src/init.ts`), so removal is symmetric with how `init` writes â€” strips only coldstart-owned markers (hook entries, `@coldstart.md` import, `AGENTS.md` block, `[mcp_servers.coldstart]` table, fully-owned files `coldstart.md` / `.cursor/rules/coldstart.mdc`), never user content in shared files. Sweeps all four clients unconditionally (init records none), prunes emptied arrays/dirs. Notebook KEPT by default (committed user data); `--purge` also deletes `.coldstart/notebook/` + its git plumbing. Idempotent (second run = all-absent, exit 0).
