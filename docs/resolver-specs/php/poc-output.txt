================================================================================
NOTES / CAVEATS DISCOVERED DURING RUN
================================================================================

1. Monica is Laravel 12. It has NO `app/Http/Kernel.php` (middleware aliases
   moved to `bootstrap/app.php` via `->withMiddleware()`). The middleware-alias
   resolution rule in the spec is therefore unverified on real data here; it
   stays deferred (matches spec's priority ordering).

2. Monica's only Blade files are 5 email partials, all using the `mail::`
   vendor namespace. The Blade resolver can't be more thoroughly exercised
   without a Laravel app that ships first-party views. If we want a stronger
   Blade test, suitable repos include `koel/koel` or older versions of Monica
   (pre-Inertia). For this spec, the resolver logic is verified on synthetic
   paths (existsSync over `resources/views/<path>`).

3. The Symfony demo's `services.yaml` only contains the `App\: resource: '../src/'`
   glob — no explicit `App\Foo: { class: ... }` lines. The
   `symfony-services-explicit` category is therefore exercised in the code
   path but has zero rows in the test output. The hand-rolled YAML scanner
   is fragile (regex over `[A-Za-z_\\]` keys). For production a real YAML
   parser is recommended; until then, this covers the common case.

4. `laravel-container-make` resolves 46/59 (78%) on Monica. The 13 unresolved
   are all `Illuminate\...` / `Symfony\...` / `League\...` framework FQCNs.
   No false positives identified. Same-namespace fallback (resolve a bare
   `Foo` to `<current namespace>\Foo` when no `use` matches) is implemented
   and improves resolution rate substantially — without it, many `app(X::class)`
   calls in deep `App\Domains\...` subtrees would fail.

5. Symfony Route attribute resolution depends on the file's
   `use Symfony\Component\Routing\Attribute\Route;` import. The POC currently
   matches by ATTRIBUTE NAME TEXT alone (`Route`), not by resolved FQCN. This
   is a small false-positive risk: any user-defined attribute also named
   `Route` would match. Acceptable risk for v1; spec describes the FQCN
   route as future work.
