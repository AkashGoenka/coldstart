FP1 — `arches.log` in LOGGING (arches/settings.py:462)
  This is a **filename string** (`os.path.join(ROOT_DIR, "arches.log")`)
  matched because it happens to contain a dot and pass the dotted-name regex.
  The strip-last-segment fallback finds `arches/__init__.py` and emits a
  spurious edge.
  Mitigation options for production:
    - In LOGGING context, scope dotted-string extraction to only the keys
      `"class"` and `"()"` (factory) inside handler/filter dicts. Reject
      string values keyed by `"filename"`, `"address"`, `"host"`, etc.
    - Reject any string that ends in a common file-extension token
      (`.log`, `.txt`, `.json`, `.sqlite3`).
